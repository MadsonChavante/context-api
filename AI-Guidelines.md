# AI Coding Guidelines for Context-API Project

## Comment Policy
- **NO COMMENTS** should be generated in any code files
- All existing comments have been removed from the codebase
- Future code contributions must not include any comments
- Exception: Only Javadoc comments for public APIs are allowed if strictly required by framework/tooling

## File Types Affected
- Java (.java) files
- XML configuration files
- Properties files
- Markdown documentation
- Any other source code files

## Enforcement
- This rule applies to all AI interactions with this project
- Violations should be corrected immediately
- Code reviews should verify absence of unnecessary comments


## Rationale
- Cleaner, more readable code
- Reduced noise in diffs
- Self-documenting code preferred over comments
- Follows modern clean code practices
